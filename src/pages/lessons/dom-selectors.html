Sélecteurs DOM en JavaScript

Le Document Object Model (DOM) est une interface de programmation pour les documents HTML et XML. Il représente la structure du document sous forme d'arborescence, où chaque nœud est un objet représentant une partie du document.

En JavaScript, nous utilisons les sélecteurs DOM pour accéder à ces nœuds et manipuler le contenu, la structure et le style des documents web.

Méthodes de Sélection d'Éléments

1. getElementById()

Sélectionne un élément par son attribut id. Il retourne un seul élément ou null si aucun élément n'est trouvé. L'id doit être unique dans le document.

<div id="monElement">Ceci est un élément.</div>
const element = document.getElementById('monElement');
console.log(element.textContent); // Affiche: Ceci est un élément.

2. getElementsByClassName()

Sélectionne tous les éléments qui ont une classe CSS spécifique. Elle retourne une HTMLCollection (collection d'éléments HTML "live").

<div class="maClasse">Un</div>
<div class="maClasse">Deux</div>
const elements = document.getElementsByClassName('maClasse');
console.log(elements.length); // Affiche: 2
console.log(elements[0].textContent); // Affiche: Un

3. getElementsByTagName()

Sélectionne tous les éléments d'un nom de balise donné (par exemple, div, p, a). Elle retourne également une HTMLCollection.

<p>Paragraphe 1</p>
<p>Paragraphe 2</p>
const paragraphes = document.getElementsByTagName('p');
console.log(paragraphes.length); // Affiche: (nombre de paragraphes dans le document)

4. querySelector()

Retourne le premier élément qui correspond à un sélecteur CSS spécifié (ex: .maClasse, #monElement, p). C'est une méthode très flexible.

<div class="conteneur">
  <p>Premier paragraphe.</p>
  <p class="special">Paragraphe spécial.</p>
</div>
const premierP = document.querySelector('p');
console.log(premierP.textContent); // Affiche: Premier paragraphe.
const specialP = document.querySelector('.special');
console.log(specialP.textContent); // Affiche: Paragraphe spécial.

5. querySelectorAll()

Retourne tous les éléments qui correspondent à un sélecteur CSS spécifié. Elle retourne une NodeList (collection de nœuds non "live").

<ul>
  <li class="item">Item 1</li>
  <li class="item">Item 2</li>
</ul>
const items = document.querySelectorAll('.item');
items.forEach(item => console.log(item.textContent));
// Affiche:// Item 1// Item 2

HTMLCollection vs. NodeList

HTMLCollection est "live", ce qui signifie qu'elle se met à jour automatiquement si le DOM change. Les méthodes comme getElementsByClassName et getElementsByTagName retournent des HTMLCollection.

NodeList est généralement statique (non "live"), sauf pour childNodes. querySelectorAll retourne une NodeList. Les NodeList peuvent être itérées avec forEach, ce qui n'est pas directement possible avec HTMLCollection sans la convertir en tableau (par exemple, avec Array.from()).

Modification d'Attributs et de Styles

Une fois qu'un élément est sélectionné, vous pouvez modifier ses attributs et ses styles.

<button id="myButton">Cliquez-moi</button>
const button = document.getElementById('myButton');
// Modification du texte
button.textContent = "Je suis cliqué!";
// Modification d'un attribut (e.g., id, class, src)
button.setAttribute('data-status', 'clicked');
// ou directement pour les classes:
button.classList.add('btn-success');
button.classList.remove('btn-primary');
// Modification de style en ligne
button.style.backgroundColor = '#10B981'; // Tailwind green-500
button.style.color = 'white';
button.style.padding = '1rem 2rem';
Conseil : Pour des manipulations de style plus complexes ou pour maintenir la séparation des préoccupations, préférez manipuler les classes CSS (classList.add(), classList.remove(), classList.toggle()) plutôt que de définir directement les styles en ligne avec element.style.
